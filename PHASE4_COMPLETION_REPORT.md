## 🎯 规则评估算法

### 评分逻辑
```javascript
function evaluateTermWithRules(term, frequency, docCount, domain) {
  // 规则 1: 长度检查
  if (term.length < 2) return score=1 (stopword);
  
  // 规则 2: 停用词检查
  if (stopwords.includes(term)) return score=1;
  
  // 规则 3: 英文术语检查
  if (/^[a-zA-Z]+$/.test(term) && frequency >= 3) 
    return score=4 (related);
  
  // 规则 4: 领域特异性检查
  if (hasDomainKeyword && frequency >= 3) 
    return score=5 (core);
  else if (hasDomainKeyword) 
    return score=4 (related);
  
  // 规则 5: 频率检查
  if (frequency >= 5 && docCount >= 2) 
    return score=4 (related);
  
  return score=3 (general);
}
```

## 💡 LLM 评估的潜在价值

虽然规则评估已足够有效，但 LLM 评估在以下场景仍有价值：

### 1. 边界案例判断
- **场景**: 术语"智能"（跨领域通用）
- **规则评估**: 3 分（general）
- **LLM 评估**: 可能识别为 4 分（在"人工智能"上下文中）

### 2. 新领域探索
- **场景**: 全新领域（无预定义关键词）
- **规则评估**: 无法识别领域特异性
- **LLM 评估**: 可基于语义理解判断

### 3. 术语关系识别
- **场景**: "LoRA"vs"LoRA 微调"vs"QLoRA"
- **规则评估**: 视为独立术语
- **LLM 评估**: 可识别层次关系

## 🚀 优化建议

### 方案 1: 规则增强（推荐）✅
在现有规则基础上增强：
- [ ] 扩展领域关键词库（覆盖更多领域）
- [ ] 添加术语组合识别（如"X+Y"型术语）
- [ ] 引入词性标注辅助判断

**优势**: 保持速度和稳定性，成本为零

### 方案 2: 混合评估 ⚖️
规则为主，LLM 为辅：
- 规则评估 score >= 4: 直接通过
- 规则评估 score = 3: 可选 LLM 复评
- 规则评估 score <= 2: 直接淘汰

**优势**: 平衡速度和准确性  
**劣势**: 增加复杂度和成本

### 方案 3: 离线 LLM 评估 📦
定期（如每周）用 LLM 评估一次：
- 批量评估所有术语
- 更新领域关键词库
- 优化规则阈值

**优势**: 低成本，持续优化  
**劣势**: 非实时
