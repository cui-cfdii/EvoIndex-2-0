# EvoIndex-CN 项目改名说明

**日期**: 2026-03-14 10:47  
**原项目名**: PageIndex-CN  
**新项目名**: EvoIndex-CN

---

## 🎯 改名原因

### 1. 算法已演化成全新架构

**原 PageIndex 算法**（2026-03-08 至 2026-03-12）:
- 基于 Markdown 章节结构的检索树
- 依赖文档格式
- 无法跨文档聚合
- 召回率：82.2%

**EvoIndex-CN 算法**（2026-03-14 至今）:
- 基于术语词典 + 倒排索引
- 不依赖文档格式
- 跨文档术语聚合
- 自进化能力（每周更新）
- 召回率：100.0%

**结论**: 算法已完全重构，继续使用"PageIndex"易造成混淆

---

### 2. 突出自进化（Evolution）特性

**Evo** = **Evo**lution（自进化）

**核心特性**:
- ✅ 从互联网自动学习新领域知识
- ✅ 每周更新术语词典
- ✅ 动态优化检索权重
- ✅ 召回率验证闭环

**改名意义**: 突出项目的**自进化能力**，与原 PageIndex 形成明显区分

---

### 3. 品牌独立性

**原项目**: PageIndex（微软 GraphRAG 思想的实现）

**新项目**: EvoIndex-CN（独立创新的自进化检索系统）

**优势**:
- ✅ 避免与微软 PageIndex 混淆
- ✅ 建立独立品牌形象
- ✅ 突出中文优化（CN）特性

## 🎯 新名称释义

### EvoIndex-CN

**Evo** = Evolution（自进化）
- 持续学习新领域知识
- 每周自动更新
- 动态优化

**Index** = 索引
- 术语词典检索
- 倒排索引
- 跨文档聚合

**CN** = Chinese（中文优化）
- jieba 分词
- 领域自适应
- 中文术语优化

**完整含义**: **自进化中文术语索引系统**

---

## 📊 品牌定位

**目标用户**:
- 中文知识库检索需求
- 需要持续更新的知识库
- 高召回率要求（>=92%）

**核心价值**:
1. **自进化**: 持续学习，无需手动维护
2. **高召回**: 100% 召回率（标准查询）
3. **零成本**: 规则评估，无需 LLM
4. **领域自适应**: 多领域扩展能力

**竞争差异化**:
- vs PageIndex: 自进化能力
- vs GraphRAG: 零 LLM 依赖
- vs 传统检索: 术语词典 + 动态更新
