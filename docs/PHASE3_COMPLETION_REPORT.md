## 🎯 实现功能

### 1. 触发事件类型（5 种）

| 类型 | 标识 | 触发条件 | 说明 |
|------|------|---------|------|
| **低匹配度** | `low_match` | 匹配度 < 80% | 查询与现有术语匹配度低 |
| **新领域** | `new_domain` | 领域不存在 | 检测到新的专业领域 |
| **频繁查询** | `frequent_query` | ≥3 次/周 | 同一查询频繁出现 |
| **用户兴趣** | `user_interest` | 置信度≥0.5 | 用户高兴趣领域 |
| **低召回率** | `low_recall` | 召回率 < 92% | 检索结果召回率低 |

---

### 2. 触发器配置

```javascript
const config = {
  matchThreshold: 0.8,        // 匹配度阈值
  recallThreshold: 0.92,      // 召回率阈值
  frequentQueryThreshold: 3,  // 频繁查询阈值（次/周）
  enabled: true,              // 是否启用
  logEvents: true             // 是否记录事件
};
```

---

### 3. 触发流程

```
用户查询
  ↓
意图识别 → 领域检测
  ↓
匹配度计算
  ↓
触发条件检测
├─ 低匹配度？→ 触发学习
├─ 新领域？→ 触发学习
├─ 频繁查询？→ 触发学习
└─ 用户兴趣？→ 触发学习
  ↓
记录触发事件
  ↓
启动学习流程
```
